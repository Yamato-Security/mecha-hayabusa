Incident Forensic Report

1. Executive Summary

From 2022-08-09 04:52 UTC to 2022-08-09 15:20 UTC, a targeted multi-stage cyber attack was confirmed across the corp.net domain environment. The attacker leveraged spearphishing emails containing a double-extension malware (Security_Protocol.doc.exe) to compromise four workstations, deployed Cobalt Strike and Empire post-exploitation frameworks for command and control, executed Mimikatz for credential theft, and used Group Policy Objects to disable Windows Defender and downgrade NTLM authentication across all domain hosts. The attack demonstrated full kill chain coverage with evidence of active financial data exfiltration targeting banking and accounting files.

8. Lateral Movement Analysis

Propagation Path

Lateral Movement Event Details

Direct host-to-host lateral movement via SMB/RDP/WinRM was not explicitly detected between individual hosts. However, the attack progression indicates:

  1. IT-03 → Domain: Mimikatz credential harvesting provided domain credentials
  2. Domain → IT-02: CobaltStrike beacon installed as a service (requires admin access, likely using stolen credentials)
  3. Phishing → ACC workstations: Four users opened the same malicious attachment nearly simultaneously, suggesting a bulk email campaign
  4. ACC-01: RDP tunnel (localhost:127.0.0.1) established for persistent graphical access
  5. ACC-09/ACC-03 → C2: Active C2 communication via msxsl.exe to secure.n-document.com
  6. Domain Controller → ALL: GPO deployment indicates domain admin compromise

The absence of traditional lateral movement (PsExec, WMI, WinRM) between hosts suggests the attacker relied on phishing for workstation access and credential abuse for server access.

Recommended Additional Investigation

  1. Domain Controller logs: Collect and analyze DC event logs to identify GPO modifications, Kerberos ticket operations, and admin account usage
  2. Email gateway logs: Correlate the Security_Protocol.doc.exe phishing email - identify sender, additional recipients, and delivery timestamps
  3. Network traffic analysis: Examine firewall/proxy logs for all connections to secure.n-document.com (duration, volume, additional endpoints)
  4. Memory forensics: Capture memory from IT-02 (CobaltStrike beacon) and IT-03 (Mimikatz) to extract full beacon configuration and harvested credentials
  5. crx.dll analysis: Retrieve and analyze the downloaded DLL from ACC-03 (C:\Temp\crx.dll) for capabilities and additional IOCs
  6. AA_V3.hta analysis: Analyze the HTA persistence payload from IT-02 (C:\Users\Gail.Goff\Documents\AA_V3.hta)
  7. Timeline gap (07:00-12:00 UTC): Investigate what the attacker did during the 5-hour gap before GPO deployment - likely privilege escalation to Domain Admin occurred during this period
  8. Financial data impact: Determine whether banking/accounting files were actually exfiltrated based on the targeted file searches

Containment and Recovery Recommendations

  1. Immediate: Isolate all 9 affected hosts from the network. Disable compromised accounts (Chad.Chan, Anna.Ward, Albert.Willoughby, Jean.Owen, Gail.Goff, Julia.Jones)
  2. Credential Reset: Reset passwords for ALL domain accounts, particularly Domain Admins. Revoke and reissue Kerberos tickets (krbtgt double reset)
  3. C2 Blocking: Block secure.n-document.com at DNS/firewall/proxy level. Block all IOC file hashes
  4. Service Removal: Remove CobaltStrike service (b8b8705) from IT-02. Remove scheduled task "AA" from IT-02. Delete ftg.exe from IT-03, msxsl.exe and crx.dll from ACC-03/ACC-09
  5. GPO Remediation: Revert malicious GPO changes - re-enable Defender real-time monitoring and restore NTLM settings
  6. Sysmon Restoration: Reinstall Sysmon across all hosts for continued monitoring
  7. Enhanced Monitoring: Deploy EDR with behavioral detection. Monitor for re-compromise attempts using same TTPs
